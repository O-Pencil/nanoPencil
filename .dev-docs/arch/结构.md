## Workspace 配置

### pnpm-workspace.yaml

```yaml
packages:
  - 'packages/*'
```

### 依赖声明示例

**catui/package.json**:

```json
{
  "name": "@pencil-agent/nano-pencil",
  "dependencies": {
    "@catui/nanomem": "workspace:*",  // ← 本地链接
    "@mariozechner/pi-agent-core": "^0.55.3"
  }
}
```

**发布时自动替换为**:

```json
{
  "dependencies": {
    "@catui/nanomem": "^0.2.0",  // ← 实际版本
    "@mariozechner/pi-agent-core": "^0.55.3"
  }
}
```

## 构建流程

```
pnpm build
    │
    ├─→ pnpm -r build
    │       │
    │       ├─→ packages/nanomem (先构建，无依赖)
    │       │       └─→ tsc → dist/
    │       │
    │       ├─→ packages/simplify (依赖 catui 类型)
    │       │       └─→ tsc → dist/
    │       │
    │       └─→ packages/catui (依赖 nanomem)
    │               └─→ tsc → dist/
    │
    └─→ 完成！所有包构建完毕
```

## 开发工作流

### 日常开发

```bash
# 1. 安装依赖
pnpm install

# 2. 启动开发模式（监听所有包）
pnpm -r --parallel dev

# 3. 只启动 catui
pnpm dev

# 4. 只监听 nanomem 变化
pnpm dev:nanomem
```

### 添加新功能

```bash
# 1. 修改 nanomem 源码
vim packages/nanomem/src/engine.ts

# 2. 自动重新编译 (watch 模式)
# → 输出到 packages/nanomem/dist/

# 3. catui 自动检测变化 (workspace 链接)
# → 无需重新安装，直接生效
```

### 发布流程

```bash
# 1. 添加 changeset
pnpm changeset
# > 选择要发布的包 (nanomem 或 catui)
# > 填写变更摘要

# 2. 更新版本号
pnpm changeset version
# > 自动更新 package.json 版本
# > 自动更新 workspace 依赖版本

# 3. 构建
pnpm build

# 4. 发布
pnpm changeset publish
# > 按顺序发布所有有变更的包
```

## 优势对比

### ❌ 旧结构 (单一仓库)

```
catui/
├── core/extensions/nanomem/  (嵌入子目录)
├── core/
├── modes/
└── ...

问题：
• nanomem 无法独立使用
• 无法单独版本管理
• 测试和构建耦合
```

### ✅ 新结构 (Monorepo)

```
packages/
├── nanomem/        (独立包)
├── catui/     (主包)
└── simplify/       (可选扩展)

优势：
✓ nanomem 可独立发布和使用
✓ 清晰的包边界和职责
✓ 灵活的版本管理
✓ 独立的测试和构建
✓ workspace 自动链接
✓ 原子提交 (跨包更改)
```

## 命令速查

```bash
# 构建所有包
pnpm build

# 构建特定包
pnpm --filter nanomem build

# 启动开发模式
pnpm dev

# 清理所有构建产物
pnpm clean

# 运行测试
pnpm test

# 添加依赖到特定包
pnpm --filter catui add lodash

# 添加 workspace 依赖
pnpm --filter catui add -D @catui/nanomem@workspace:*

# 发布
pnpm changeset
pnpm changeset version
pnpm changeset publish
```
