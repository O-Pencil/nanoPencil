# Issue Usage Evaluation

> Type: Evaluation Category
> Status: Draft
> Scope: How Agent uses issues as task boundaries and validation contracts
> Purpose: Evaluate whether issue-driven work becomes bounded, executable, and verifiable

## 1. Question

Does the Agent convert issue context into a clear task boundary, implementation path, and validation result?

This category also acts as the bridge from eval suggestions to executable follow-up work.

## 2. Documents

| File | Purpose |
|------|---------|
| `workflow.md` | Issue-to-validation workflow |
| `metrics.md` | Review dimensions for issue usage |
| `cases.md` | Case log template |

## 3. Role in Eval

Eval suggestions should usually become issues before they become Agent behavior changes.

Issue usage evaluation checks whether those issues preserve:

1. Original observation.
2. Proposed suggestion.
3. Acceptance criteria.
4. Validation plan.
5. Final adoption or rejection status.
