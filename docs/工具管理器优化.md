# 工具管理器性能优化

## 问题

在 Windows 上每次启动 nanopencil 时，都会看到以下提示：
```
fd not found. Downloading...
ripgrep not found. Downloading...
```

即使这些工具已经下载并安装在 `~/.nanopencil/agent/bin/` 目录中，每次启动仍会检查并显示这些消息。

## 根本原因

`getToolPath()` 函数每次被调用时都会执行文件系统检查：
1. 检查本地工具目录是否存在该工具
2. 检查系统 PATH 中是否存在该工具
3. 每次启动都会重复这些检查

由于多个组件可能在启动时调用 `ensureTool()` 或 `getToolPath()`，导致重复的文件系统 I/O 操作。

## 解决方案

### 1. 添加缓存机制

在 `utils/tools-manager.ts` 中添加了内存缓存：

```typescript
// Cache for tool availability checks to avoid repeated filesystem checks
const toolCheckCache = new Map<string, string | null>();
```

### 2. 优化 `getToolPath()` 函数

```typescript
export function getToolPath(tool: "fd" | "rg"): string | null {
  const config = TOOLS[tool];
  if (!config) return null;

  // ✅ 先检查缓存
  if (toolCheckCache.has(tool)) {
    return toolCheckCache.get(tool) ?? null;
  }

  // 检查本地工具目录
  const localPath = join(TOOLS_DIR, config.binaryName + (platform() === "win32" ? ".exe" : ""));
  if (existsSync(localPath)) {
    toolCheckCache.set(tool, localPath);  // ✅ 缓存结果
    return localPath;
  }

  // 检查系统 PATH
  if (commandExists(config.binaryName)) {
    toolCheckCache.set(tool, config.binaryName);  // ✅ 缓存结果
    return config.binaryName;
  }

  // ✅ 缓存负面结果（工具不存在）
  toolCheckCache.set(tool, null);
  return null;
}
```

### 3. 更新 `ensureTool()` 函数

下载成功后更新缓存：

```typescript
try {
  const path = await downloadTool(tool);
  toolCheckCache.set(tool, path);  // ✅ 下载后更新缓存
  if (!silent) {
    console.log(chalk.dim(`${config.name} installed to ${path}`));
  }
  return path;
} catch (e) {
  // ...
}
```

### 4. 新增辅助函数

```typescript
/**
 * 清除工具检查缓存（用于强制重新检查）
 */
export function clearToolCheckCache(): void {
  toolCheckCache.clear();
}

/**
 * 获取缓存统计信息（用于调试）
 */
export function getToolCheckCacheStats(): { 
  size: number; 
  entries: Array<{ tool: string; path: string | null }> 
} {
  return {
    size: toolCheckCache.size,
    entries: Array.from(toolCheckCache.entries()).map(([tool, path]) => ({ tool, path })),
  };
}
```

## 性能提升

### 优化前
- 每次启动调用 `getToolPath()` 2-4 次
- 每次调用执行 2-3 次文件系统检查
- 总计：4-12 次文件系统 I/O 操作

### 优化后
- 首次调用：2-3 次文件系统检查（未命中缓存）
- 后续调用：0 次文件系统检查（命中缓存）
- 总计：2-3 次文件系统 I/O 操作

**性能提升**: 约 75% 的文件系统 I/O 减少

## 注意事项

### 缓存生命周期

- 缓存在进程生命周期内有效
- 进程退出后缓存自动清除
- 下次启动重新建立缓存（但只检查一次）

### 何时清除缓存

一般情况下不需要手动清除缓存，但如果需要：
- 工具被外部程序删除
- 需要强制重新检查工具可用性
- 调试工具查找逻辑

可以通过以下方式清除缓存（未来可能添加 `/reload-tools` 命令）：
```typescript
import { clearToolCheckCache } from './utils/tools-manager.js';
clearToolCheckCache();
```

### 离线模式

离线模式 (`PI_OFFLINE=1`) 不受影响：
- 如果工具已缓存，返回缓存的路径
- 如果工具未缓存且在离线模式下，跳过下载

## 测试

### 验证缓存工作

```typescript
import { getToolPath, getToolCheckCacheStats } from './utils/tools-manager.js';

// 首次调用 - 检查文件系统
console.log(getToolPath('fd'));  // 'C:\...\.nanopencil\agent\bin\fd.exe'

// 后续调用 - 使用缓存
console.log(getToolPath('fd'));  // 从缓存返回

// 查看缓存状态
console.log(getToolCheckCacheStats());
// { size: 1, entries: [ { tool: 'fd', path: '...' } ] }
```

### 验证负面结果缓存

```typescript
// 清除缓存
clearToolCheckCache();

// 检查不存在的工具
console.log(getToolPath('rg'));  // null（如果未安装）
console.log(getToolCheckCacheStats());
// { size: 1, entries: [ { tool: 'rg', path: null } ] }

// 再次检查 - 使用缓存
console.log(getToolPath('rg'));  // null（从缓存，不检查文件系统）
```
